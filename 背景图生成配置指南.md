# 背景图生成功能配置指南

## 问题描述
点击"开始学习"按钮后，背景图没有变成硅基流动文生图模型生成的图片。

## 问题原因
1. **API密钥未配置**：硅基流动API需要有效的API密钥才能调用文生图模型
2. **环境变量文件缺失**：项目中只有`.env.example`，没有实际的`.env`文件

## 解决方案

### 步骤1：获取硅基流动API密钥

1. 访问硅基流动官网：https://cloud.siliconflow.cn/
2. 注册并登录账号
3. 在控制台中获取API密钥

### 步骤2：配置环境变量

1. 在 `backend` 目录下创建 `.env` 文件（已自动创建）
2. 编辑 `backend/.env` 文件，将 `SILICONFLOW_API_KEY` 替换为您的实际API密钥：

```env
# 硅基流动API配置
SILICONFLOW_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxx
```

### 步骤3：重启后端服务

```bash
cd backend
npm install dotenv  # 如果未安装dotenv
node server.js
```

## 功能说明

### 工作流程
1. 用户打开诗词详情页
2. 页面自动调用 `/api/ai/image/pregenerate` 接口预生成背景图
3. 后端调用硅基流动API生成图片
4. 通过Socket.io实时推送图片生成状态
5. 用户点击"开始学习"按钮时，显示生成的背景图

### Socket事件
- `image-generate-pending`：图片生成中
- `image-generate-success`：图片生成成功
- `image-generate-fail`：图片生成失败

### 缓存机制
- 生成的图片会缓存30天
- 相同诗词会复用缓存的图片
- 缓存存储在内存中（Map对象）

```

```

### 2. 检查前端控制台


## 常见问题

### Q1: 提示"API密钥未配置"
**A**: 请确保 `backend/.env` 文件中已正确配置 `SILICONFLOW_API_KEY`，并重启后端服务。

### Q2: 图片生成失败
**A**: 可能的原因：
- API密钥无效或过期
- API调用次数超限
- 网络连接问题
- Prompt内容不符合API要求

解决方法：
1. 检查API密钥是否有效
2. 查看硅基流动控制台的用量统计
3. 检查后端日志中的错误信息

### Q3: 图片加载很慢
**A**: 图片生成需要时间（通常10-30秒），这是正常现象。前端会显示"诗中有画，画境渐生..."的加载提示。

### Q4: 使用默认背景
**A**: 如果图片生成失败，系统会自动使用默认的古风背景，不影响学习功能的使用。

## 技术细节

### API调用参数
```javascript
{
  model: 'Kwai-Kolors/Kolors',
  prompt: '根据古诗词《标题》（作者：作者）的内容生成一幅中国风图像...',
  image_size: '1440x720',
  batch_size: 1,
  num_inference_steps: 20,
  guidance_scale: 7.5
}
```

### 图片格式
- 格式：PNG
- 编码：Base64
- 尺寸：1440x720
- 数据URL：`data:image/png;base64,{base64数据}`
